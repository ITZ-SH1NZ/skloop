A well-structured page helps browsers and developers understand how the content is arranged.</p>\n\n<p>Semantic HTML means using elements that clearly describe the purpose of the content instead of using generic tags like <code>&lt;div&gt;</code> for everything.</p>\n\n<h2>Basic HTML Structure</h2>\n\n<p>Every HTML page follows a simple structure:</p>\n\n<pre><code>&lt;!DOCTYPE html&gt;\n&lt;html&gt;\n&lt;head&gt;\n  &lt;title&gt;Page Title&lt;/title&gt;\n&lt;/head&gt;\n&lt;body&gt;\n  &lt;h1&gt;Main Heading&lt;/h1&gt;\n  &lt;p&gt;This is a paragraph.&lt;/p&gt;\n&lt;/body&gt;\n&lt;/html&gt;</code></pre>\n\n<p><strong><code>&lt;html&gt;</code></strong> – The root element that contains the whole webpage.</p>\n\n<p><strong><code>&lt;head&gt;</code></strong> – Contains information about the page such as the title.</p>\n\n<p><strong><code>&lt;body&gt;</code></strong> – Contains the visible content of the webpage.</p>\n\n<h2>Common Semantic Elements</h2>\n\n<p><strong><code>&lt;header&gt;</code></strong> – The top section of a page or section.</p>\n\n<p><strong><code>&lt;nav&gt;</code></strong> – Contains navigation links.</p>\n\n<p><strong><code>&lt;main&gt;</code></strong> – The main content of the page.</p>\n\n<p><strong><code>&lt;section&gt;</code></strong> – A group of related content.</p>\n\n<p><strong><code>&lt;article&gt;</code></strong> – Independent content like a blog post.</p>\n\n<p><strong><code>&lt;footer&gt;</code></strong> – The bottom section of a page.</p>\n\n<h2>Why It Matters</h2>\n\n<ul>\n  <li>Makes webpages easier to understand.</li>\n  <li>Helps search engines read your content.</li>\n  <li>Makes code cleaner and easier to maintain.</li>\n</ul>"
